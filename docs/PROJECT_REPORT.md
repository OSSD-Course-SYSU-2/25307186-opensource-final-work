# 开源学习助手项目报告

## 一、选题背景

本课程大作业要求使用 DevEco Studio 开发一个 HarmonyOS 小软件并提交至 GitHub。结合《开源软件开发技术》课程主题，本项目选择实现“开源学习助手”，用移动端小应用的形式记录学习待办、了解许可证差异并熟悉开源贡献流程。

## 二、需求分析

- 需要一个清晰的 HarmonyOS 应用工程，可以被 DevEco Studio 打开运行；
- 应用需要有明确主题，而不是空模板；
- 应用需要包含基础交互，体现 ArkTS 与 ArkUI 的使用；
- 仓库需要有 README、许可证和项目说明，满足开源项目基本规范。

## 三、功能设计

1. **总览页**：展示学习进度、项目定位、核心功能和技术栈；
2. **待办页**：支持添加任务、标记完成、撤销完成、删除任务；
3. **许可证页**：对比 MIT、Apache-2.0、GPL-3.0；
4. **贡献页**：展示从 Fork 到 Merge 的开源协作流程。

## 四、技术实现

项目采用 HarmonyOS Stage 模型，入口为 `EntryAbility`。主页面 `Index.ets` 使用 ArkUI 声明式组件构建界面，通过 `@State` 管理当前标签页、任务列表、输入框内容和当前许可证选择。任务列表变更时通过重新赋值触发界面刷新。

## 五、开源规范

仓库补充了：

- `README.md`：说明项目背景、运行方式和目录结构；
- `LICENSE`：使用 MIT License；
- `.gitignore`：排除构建产物、依赖目录和 IDE 临时文件。

## 六、总结

本项目完成了一个可展示的 HarmonyOS 小应用，覆盖课程要求中的 DevEco Studio 开发、GitHub 提交和开源项目基本文档。后续可继续增加本地持久化、任务分类、学习资料链接等功能。
